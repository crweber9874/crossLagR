3  Latent Change Score Models

Whereas the previous chapter established dynamic lagged and cross lagged effects in level form (the CLPM and its bias under unit heterogeneity), this chapter turns to the change representation. By “differencing” the observed scores, this effectively absorbs all time-invariant heterogeneity, including stable between-person differences.

The Latent Change Score Model (LCSM) parameterizes dynamics differently as differences across waves (McArdle 2009; Kim and Steiner 2021). The logic is to remove out any stable respondent effects by simply calculating the wave to wave difference. The LCSM uses a measurement equation that separates latent true scores from unique factors, and a dynamic equation that maps lagged true scores onto latent change scores via a proportional self-feedback parameter (\(\beta\), i.e. ar_x) and a coupling parameter (\(\omega\), i.e. cl_xy). Finally, a second-order constant change factor can be specified which accumulates over waves, though requires 4+ waves for identification. This term captures stable, person-specific trajectories that are not accounted for by differencing.

3.1 The LCSM

Figure 3.1 sketches the bivariate LCSM with a constant-change factor, in the same DAG style introduced in the previous chapter. Each observed score is decomposed as \(X_t = X_{t-1} + \Delta x_t\); each change score is regressed on prior levels via proportional change (\(\beta\)) and coupling (\(\omega\)), and accumulates a person-specific constant (\(\Delta_x\), \(\Delta_y\)). The detailed measurement-model version with multiple indicators is reproduced in the appendix as Figure 3.2.

3.2 estimateLChange(): Generated lavaan Syntax

estimateLChange() returns a lavaan model string with the unified labels: ar_x, ar_y are proportional self-feedback (the effective autoregressive parameter is \(1+AR()\)); cl_xy, cl_yx are coupling (cross-lagged) parameters; A_x, A_y are accumulating constant-change factors; u_var_x, u_var_y are unique-factor (measurement-error) variances. The function returns syntax that can be passed directly to lavaan::lavaan().

3.2.4 Bivariate LCSM with change-to-change effects

Requires \(\geq 4\) waves; adds parameters phi_x, phi_y (change AR) and phi_cl_x, phi_cl_y (change cross-lag).

Unfortunately, there is not a “level” analogue to this specification. The change-to-change parameters capture change on change itself. In particular, \(\phi_x\) is the effect of prior change in \(x\) on subsequent change in \(x\), while \(\phi_{cl_x}\) is the effect of prior change in \(y\) on current/subsequent change in \(x\).

3.3 simLChange(): Simulating from the LCSM

simLChange() simulates data under user specified true values. Arguments mirror the estimator syntax: beta_x/beta_y (proportional change), omega_x/omega_y (coupling), constant_mean_x/constant_mean_y and latent_variance_x/latent_variance_y (constant-change factor mean and variance), and estimate_constant_change to toggle accumulating change.

sim <- simLChange(
  waves                    = 5,
  variable_type            = "bivariate",
  beta_x                   = -0.7,
  beta_y                   = -0.7,
  omega_x                  = 0.5,

3.4 Algebraic Equivalence: LCSM = CLPM in Change Form

Without a constant-change factor and without measurement error, the LCSM dynamic equation \(\Delta x_t = \beta_x x_{t-1} + \omega_x y_{t-1} + u\) rearranges to \(x_t = (1+\beta_x) x_{t-1} + \omega_x y_{t-1} + u\) — exactly the CLPM regression on the level scores. The CLPM should recover \(1+\beta\) for autoregressives and \(\omega\) for cross-lags. I do a lot of this throughout the document – it’s useful to see the equivalence across models

  1. Measurement error. simLChange() defaults indicator_variance_x = indicator_variance_y = 0.5, putting noise on the observed scores. The CLPM has no latent measurement model, so its coefficients are attenuated by errors-in-variables. The LCSM separates true scores from indicators and is immune.
  2. Constant change. When the accumulating factor is on, each unit carries a stable trajectory; the CLPM treats this as wave-specific variation and biases AR up / CL down.

The simulated data – under the latent change specification – leads to estimates for the CLPM which mirror the generating parameters.

clean <- simLChange(
  waves                    = 5,
  variable_type            = "bivariate",
  beta_x                   = -0.2,
  beta_y                   = -0.2,
  omega_x                  = 0.20,
  omega_y                  = 0.55,
  indicator_variance_x     = 0.001,
  indicator_variance_y     = 0.001,
  estimate_constant_change = FALSE,
  sample.nobs              = 1000
)

3.6 Summary

The LCSM is a powerful framework for modeling dynamics in change form. By differencing observed scores, it effectively absorbs all time-invariant heterogeneity, including stable between-person differences. The estimateLChange() function generates lavaan syntax for a variety of LCSM specifications, while simLChange() simulates data.

There is a clear algebraic equivalence between the LCSM without constant change and the CLPM in change form. However, the presence of measurement error and/or constant change leads to bias in the CLPM estimates, while the LCSM can recover the true parameters.
